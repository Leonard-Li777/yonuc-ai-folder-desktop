# Руководство по Виртуальному Каталогу

## Обзор

`.VirtualDirectory` - это автоматически генерируемый виртуальный каталог данного приложения, используемый для отображения структуры файлов после интеллектуальной организации. Он поддерживает однозначное соответствие файлам в исходном каталоге, но использует интеллектуальное именование.

## Назначение

Основная цель этого виртуального каталога - позволить пользователям предварительно просматривать результаты организации файлов без фактического перемещения или копирования исходных файлов.
Когда вы будете удовлетворены окончательным результатом, вы можете нажать "Организовать Реальный Каталог", чтобы организовать реальный каталог в соответствии со структурой файлов .VirtualDirectory, после чего данное приложение удалит каталог .VirtualDirectory.

## Технические Принципы

### Технология Жестких Ссылок

Файлы в виртуальном каталоге генерируются с использованием технологии жестких ссылок. Жесткие ссылки можно просто понимать как ссылки или псевдонимы файлов, со следующими характеристиками:

1. Не занимается дополнительное физическое дисковое пространство
2. Совместно использует те же блоки данных с исходным файлом
3. Изменения в файлах с жесткими ссылками синхронизируются с исходным файлом
4. Удаление файла с жесткой ссылкой не влияет на исходный файл
5. При удалении исходного файла необходимо удалить файл с жесткой ссылкой (данное приложение активно обнаруживает удаления файлов в реальном каталоге и соответствующим образом удаляет файлы с жесткими ссылками в виртуальном каталоге.)

### Разница с Ярлыками

Хотя жесткие ссылки в некоторой степени похожи на ярлыки, между ними существуют важные различия:

| Характеристика | Ярлыки | Жесткие Ссылки |
|----------------|--------|----------------|
| Уровень Файловой Системы | Только концепция Windows | Функция файловой системы операционной системы |
| Занимаемое Пространство | Минимальное (только метаданные) | Без дополнительного пространства |
| Удаление Исходного Файла | Ярлык становится недействительным | Жесткая ссылка все еще может получить доступ к содержимому файла |
| Изменение Содержимого | Не влияет на исходный файл | Синхронизируется со всеми ссылками |
| Поддержка Разных Томов | Поддерживается | Ограничено той же файловой системой |
